## 升級規則
- 討論 agent 升級時，先以核心 6 模組為唯一正式主線。
- pack / adapter / audit / archive 不得混入核心架構判斷。
- 若核心 contract 未變，不得先改 phase-specific heuristics。
## 已落地：problem_layer root-cause signal
- `map_building_pack.py` 每個 `_append_decision` 必須寫入 `problem_layer`：`data`、`parameter`、`framework`。
- `candidate_pool.py` 優先使用明確 `problem_layer`，缺值才允許文字推斷。
- `current_state.py` 的 `problem_layer_signal` 只用診斷候選計算 dominant layer。
- `arbiter.py` 可依 dominant layer 選擇下一步：資料層回上游、參數層檢查訓練/匯出、框架層切換策略。
- 2026-04-30 replay `latest_train_complete` / `latest_export_complete` 後，兩者 dominant layer 均為 `parameter`，符合目前阻塞點。

## Learning Curve 閉環（2026-05-01）
- `coordinator.py` 每輪寫出 `outcome_feedback.json` 後，會同步產生 `learning_curve.json`。
- `outcome_feedback.json` 現在包含 preliminary outcome label：`decision_useful`、`human_override`、`wasted_run`、`repeated_problem`、`critical_bad_release`、`token_cost_estimate`。
- `candidate_pool.py` 會讀取歷史 feedback；若已有人工或後續結果標籤，優先使用 `effectiveness_rate`，否則退回 `accepted_rate` 影響 `rank_score`。
- `adapters/adaptive_threshold.py` 現在優先掃描 `outputs/phase0/*/*/outcome_feedback.json` 並重建 threshold history；只有在正式 feedback 不存在時，才回退舊 `phase0_decisions.log`。
- `arbiter.py` 的 hold-path 現在會正式使用 `candidate_pool` 的排序結果；同 layer 有多個候選時，優先選 `rank_score` 較高者，而不是固定硬編碼 `VAL-001 / REC-001`。
- `ProductionParamGate` 現在只在產出可執行 rerun 參數時 `approved=true`；若只是維持現況或缺上游資料，正式 gate 狀態為 `hold_manual_review`。
- `ProductionParamGate.evaluate()` 現在會回傳正式 `gate_status / reason / sfm_profile / train_profile`，供 pack 與 audit 使用；不再允許只有 `approved=true` 的空洞 gate。
- `map_building_pack.py` 對 `ProductionParamGate` 的正式事件已分流為 `production_params_ready`、`production_params_hold`、`production_params_failed`；只有 `approved=true` 才能進入 ready path。
- `learning_curve.json` 用於判斷對話框 AI 是否可從 meta-evaluator 降為 observer-only；目前未達標，因 latest train/export replay 都仍是 `held_for_review` 且尚未標籤。
## Outcome Label CLI（2026-05-01）
- 人工標籤不得手改 JSON；必須使用 `run_phase0.py --label-feedback <outcome_feedback.json>`。
- 可標記欄位：`--decision-useful`、`--metrics-improved`、`--problem-layer-correct`、`--human-override`、`--wasted-run`、`--repeated-problem`、`--critical-bad-release`。
- bool 值接受 `true / false / none`，也接受 `yes / no`。
- CLI 會更新該 `outcome_feedback.json`，並重算同一 stage 目錄下的 `learning_curve.json`。
- 範例：`python run_phase0.py --label-feedback D:\agent_test\outputs\phase0\<run_id>\train_complete\outcome_feedback.json --decision-useful false --human-override true --wasted-run true --label-source human_review --label-note "manual review rejected"`。
